## 목표
모든 프론트엔드 기능에 대해 사용자 동선 기반 E2E 테스트 시나리오를 작성한다.
실행은 하지 않고 시나리오 파일만 작성.

## 기능별 시나리오 (파일 분리)

### e2e/tasks-list.spec.ts (15-20개)
- 각 탭 클릭 → active 스타일 + 필터링
- Priority 필터 All/High/Medium/Low → active + 필터
- 빈 탭 → "해당 상태의 태스크가 없습니다"
- 검색 → 결과 필터링
- 정렬 변경 (최신순/오래된순/우선순위/ID)
- 의존 체인 아코디언 펼침/접힘
- 순서 변경 (위/아래)
- 페이지네이션

### e2e/task-detail.spec.ts (15-20개)
- 상태 드롭다운 변경 → UI 반영
- 실행 버튼 → 로그 탭 전환 + 실행 배너
- Stop 버튼 → DELETE 요청
- done/rejected → 실행 버튼 비활성화
- failed → Retry/Done 버튼 표시
- 각 탭(Content/Scope/Cost/AI Result/로그/리뷰결과) 전환
- branch 뱃지 클릭 → 복사
- 의존 관계 표시

### e2e/notices.spec.ts (10개)
- 목록 표시
- 필터 탭 (All/Info/Warning/Error)
- 안 읽은 Notice 시각 구분
- Notice 클릭 → read 처리
- 사이드바 뱃지 숫자 감소
- 빈 상태 표시

### e2e/night-worker.spec.ts (10개)
- 설정 탭 → 유형 선택/해제
- 시작 버튼 → 로그 탭 전환
- 중지 버튼
- 예산 무제한 체크
- 종료 시간 설정

### e2e/settings.spec.ts (10개)
- API 키 입력 + 저장
- src 경로 추가/삭제
- 모델 선택
- maxParallel 변경
- Save 버튼 활성/비활성

### e2e/sidebar.spec.ts (10개)
- Tasks 정렬 (in_progress 최상단)
- Docs 접기/펼치기
- Notice 뱃지 표시
- 네비게이션 active 상태
- Night Worker 메뉴

### e2e/current-tab.spec.ts (10개)
