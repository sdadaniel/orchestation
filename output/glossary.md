# 용어 정의

프로젝트에서 사용하는 핵심 용어를 정의한다.

## 오케스트레이션 (Orchestration)

여러 개의 태스크를 정의된 순서와 의존성에 따라 조율하고 실행하는 전체 흐름 관리 방식. 본 프로젝트에서는 태스크 정의 → 실행 → 리뷰 → 완료의 파이프라인을 의미한다.

## 태스크 (Task)

작업의 최소 단위. 각 태스크는 고유 ID, 완료 조건, 의존성, 담당 브랜치 등의 메타데이터를 가지며, `docs/task/` 디렉토리에 정의된다.

## 워크트리 (Worktree)

Git의 worktree 기능을 활용하여 태스크별로 분리된 작업 디렉토리. 각 작업자는 지정된 워크트리 안에서만 코드를 수정하며, main 브랜치에 직접 영향을 주지 않는다.

## 감독관 (Orchestrator)

전체 태스크 흐름을 관리하고 조율하는 역할. 태스크 간 의존성을 확인하고, 선행 태스크 완료 후 다음 태스크를 할당하며, 작업 결과를 리뷰하고 완료 조건 충족 여부를 판단한다.

## 작업자 (Worker)

할당된 태스크를 실제로 수행하는 역할. 지정된 워크트리 안에서만 작업하며, 태스크의 완료 조건을 충족하도록 코드를 수정하고 커밋한다.

## 의존성 (Dependency)

태스크 간의 선후 관계. 선행 태스크(`depends_on`)가 완료되어야 해당 태스크를 시작할 수 있으며, 후행 태스크(`blocks`)는 현재 태스크의 완료를 기다린다.

## 완료 조건 (Acceptance Criteria)

태스크가 완료되었다고 판단하기 위한 체크리스트. 감독관이 리뷰 시 이 조건들을 기준으로 태스크의 완료 여부를 결정한다.
