너는 B2B SaaS 대시보드를 전문으로 만드는 시니어 UX 디자이너다.

현재 화면의 문제:

- 좌우 여백이 과도하여 정보 밀도가 낮다
- 카드가 불필요하게 크고 내용 대비 공간 낭비가 심하다
- 스캔이 어려운 구조 (한 눈에 상태 파악 불가)
- 실제 작업 중심 UI가 아니라 "비어있는 느낌"이 강하다

목표:

- "한 화면에서 최대한 많은 유의미한 정보"를 보여주는 것
- 빠르게 상태 파악 + 액션 가능한 UI로 개선
- 개발자 툴/운영 대시보드 수준의 밀도와 효율성

요구사항:

1. 레이아웃을 재구성해라
    - max-width 제한 (예: 1200~1400px)
    - 좌우 여백 줄이고 중앙 집중형 레이아웃 적용
    - 카드 대신 리스트/테이블 중심으로 변경

2. 정보 밀도를 높여라
    - 카드 높이 최소화
    - 한 섹션에 최소 5~10개의 데이터가 보이도록 구성
    - "배정된 태스크가 없습니다" 같은 empty state는 축소

3. 구조 개선
    - 섹션 → 그룹 → 아이템 계층 명확히
    - Sprint / Task / Plan 구분을 더 직관적으로 재배치

4. 인터랙션 추가
    - hover 시 액션 버튼 노출
    - 클릭 영역 확대
    - 빠른 액션 (edit, assign 등) inline 제공

5. UI 스타일 방향
    - Notion / Linear / Jira 스타일 참고
    - 불필요한 박스 UI 제거 (boxless / minimal card)
    - typography + spacing 중심 디자인

6. 결과물
    - 개선된 레이아웃 구조 설명
    - before / after 차이 명확히 설명
    - 컴포넌트 단위 설계 (React 기준)

"이 화면은 하루 8시간 사용하는 내부 툴이다"라고 가정하고 재설계해라

너는 단순한 UI 개발자가 아니라,
"사용 흐름과 인터랙션을 설계하는 시니어 UX 엔지니어"다.

지금 문제:

- 현재 UI는 기능은 있지만 UX 흐름이 없다
- 사용자의 행동에 따른 상태 변화가 고려되지 않았다

핵심 요구사항:

1. "사용자의 행동 → UI 상태 변화"를 반드시 설계해라

다음 시나리오를 구현해라:

- 사용자가 가운데 task 리스트를 클릭하면:
  → 해당 task가 선택된다
  → 오른쪽 상세 패널이 열린다

- 사용자가 이미 선택된 task를 다시 클릭하면:
  → 선택 해제
  → 오른쪽 패널 닫힘

- 사용자가 빈 영역(background)을 클릭하면:
  → 선택 해제
  → 오른쪽 패널 닫힘

- ESC 키를 누르면:
  → 선택 해제
  → 패널 닫힘

2. 상태 기반 UI로 설계해라

- selectedTask 상태를 기준으로 UI가 바뀌어야 한다
- UI는 항상 "상태(state)의 결과"여야 한다
- 임시 DOM 조작 금지

3. 레이아웃 동작

- 오른쪽 패널은 "조건부 렌더링"이어야 한다
- 패널이 닫히면 가운데 영역이 자동으로 확장되어야 한다

4. UX 디테일

- 선택된 row는 시각적으로 강조
- hover 상태 제공
- 클릭 가능한 영역 명확히 표시

5. 결과물

- 상태 구조 (React 기준)
- 이벤트 흐름
- 핵심 코드 (핵심 부분만)
- UX 흐름 설명

중요:
"이 정도는 당연히 되겠지"라는 가정을 절대 하지 말고
모든 인터랙션을 명시적으로 구현해라
