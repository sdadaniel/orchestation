# 프롬프트 성능 비교 테스트 스펙

## 목표

동일한 PRD(URL 단축 서비스)를 서로 다른 프롬프트로 구현하여 성능을 비교한다.

## 입력

- PRD: `docs/prd.md` (URL 단축 서비스)
- 프롬프트: `src/prompts/p{N}/prompt{N}.md`

## 디렉토리 구조

```
src/prompts/p1/

  prompt1.md        # 프롬프트 가이드 문서
  output.md         # Claude 텍스트 출력 원본
  output.json       # Claude JSON 출력 (--output-format json)
  report.md         # 아래 측정 항목을 정리한 리포트
  src/              # 코드 구현 결과물 (있는 경우)
```

## 측정 항목

### 1. 리소스 사용량 (output.json에서 추출)

| 항목                          | 설명                            |
| ----------------------------- | ------------------------------- |
| `input_tokens`                | 입력 토큰 수                    |
| `output_tokens`               | 출력 토큰 수                    |
| `cache_creation_input_tokens` | 캐시 생성 토큰 수               |
| `cache_read_input_tokens`     | 캐시 읽기 토큰 수               |
| `total_cost_usd`              | 총 비용 (USD)                   |
| `duration_seconds`            | 총 소요 시간 (초)               |
| `num_turns`                   | 에이전트 턴 수 (tool call 횟수) |

### 2. 출력 품질 - 구조 완성도

PRD에서 요구한 10개 섹션의 포함 여부를 체크한다.

| 섹션                                    | 체크 |
| --------------------------------------- | ---- |
| 1. 문제 정의                            |      |
| 2. 사용자 정의                          |      |
| 3. 핵심 기능 (MVP)                      |      |
| 4. 기능 상세 정의 (입력/출력/예외/보안) |      |
| 5-1. 아키텍처 설계                      |      |
| 5-2. 데이터 모델 (테이블/인덱스)        |      |
| 5-3. URL 생성 전략                      |      |
| 6. API 설계 (REST 스펙)                 |      |
| 7. 성능 및 확장성                       |      |
| 8. 보안 (악성URL/rate limit/abuse)      |      |
| 9. 운영 전략 (로그/모니터링/장애대응)   |      |
| 10. 개발 계획 (Sprint 기반)             |      |

- **구조 완성도 점수**: 포함된 섹션 수 / 12

### 3. 출력 품질 - 깊이 평가

각 섹션별로 깊이를 3단계로 평가한다.

| 등급         | 기준                                                             |
| ------------ | ---------------------------------------------------------------- |
| **A (상세)** | 구체적 수치, 근거 있는 기술 선택, 예외 케이스까지 포함           |
| **B (보통)** | 항목은 있으나 구체성 부족, "적절히/필요시" 같은 애매한 표현 포함 |
| **C (나열)** | 단순 키워드 나열 수준, 근거 없음                                 |

### 4. 출력 품질 - 코드 구현 (src/ 기준)

| 항목              | 설명                        |
| ----------------- | --------------------------- |
| 파일 수           | 생성된 소스 파일 개수       |
| 총 코드 라인 수   | 빈 줄/주석 제외             |
| 실행 가능 여부    | 빌드/실행이 되는지 (Y/N)    |
| 테스트 포함 여부  | 테스트 코드가 있는지 (Y/N)  |
| API 엔드포인트 수 | 구현된 REST 엔드포인트 개수 |

### 5. 효율성 지표 (계산)

| 지표      | 산출식                              |
| --------- | ----------------------------------- |
| 토큰 효율 | 구조 완성도 점수 / total_tokens     |
| 비용 효율 | 구조 완성도 점수 / total_cost_usd   |
| 시간 효율 | 구조 완성도 점수 / duration_seconds |
| 코드 밀도 | 코드 라인 수 / output_tokens        |
