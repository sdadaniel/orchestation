# 브랜치 충돌 시 예외처리

## 배경

워커가 태스크별 브랜치(worktree)에서 작업할 때, 같은 파일을 수정하는 태스크가 동시에 실행되면 merge 시점에서 충돌이 발생한다.
현재는 충돌 시 별도 예외처리가 없어 파이프라인이 멈추거나, 의존성 체인 태스크가 충돌 상태 그대로 진행될 수 있다.

## 고려 사항

- 충돌 감지: merge/rebase 실패 시 자동 감지
- 태스크 상태 처리: 충돌 발생 시 해당 태스크를 `failed` 또는 `conflict` 상태로 전환
- 알림: 대시보드에 충돌 태스크와 충돌 파일 목록 표시
- 재시도 전략: 선행 태스크 merge 후 후행 태스크 자동 rebase 또는 재실행
- scope 겹침 사전 방지: 태스크 실행 전 scope 파일이 겹치는 동시 실행 태스크가 있는지 체크

## 관련

- TASK-111: 상태 변경 시 의존성 검증 추가
- orchestrate.sh: 병렬 실행 로직
- run-worker.sh: 브랜치 생성/merge 로직
