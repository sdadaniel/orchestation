# GlobalSearch 최근 검색어 히스토리 기능 제안

## 개요

GlobalSearch 컴포넌트에 **최근 검색어 히스토리** 기능을 추가하여 반복적으로 같은 태스크나 문서를 검색하는 사용 패턴의 UX를 개선합니다.

## 현황 분석

현재 GlobalSearch는:
- ✓ 실시간 검색 필터링 지원 (task:, doc: 접두사 포함)
- ✓ 키보드 네비게이션 (↑↓, Enter)
- ✓ Cmd+K 단축키
- ✗ 최근 검색어 기록 없음
- ✗ 자주 검색하는 항목에 대한 빠른 접근

## 제안 기능

### 1. 최근 검색어 저장 (localStorage)
- 최근 검색어 5개를 `localStorage`에 저장
- 키: `globalsearch-history` (JSON 배열)
- 구조:
  ```json
  [
    { "query": "task:001", "timestamp": 1712145000000 },
    { "query": "doc:API", "timestamp": 1712144900000 }
  ]
  ```

### 2. 검색창 포커스 시 히스토리 표시
- 검색창에 포커스 + 빈 입력 상태 → 최근 검색어 목록 표시
- 드롭다운에 "Recent" 그룹 추가
- 각 항목에 시계 아이콘(⏱️) 표시로 히스토리 항목 구분

### 3. 상호작용
- 히스토리 항목 클릭 → 해당 검색어로 즉시 검색
- 검색 완료 후 → 자동으로 히스토리에 추가
- 중복 검색어 입력 → 기존 항목을 맨 앞으로 이동 (LRU)
- "Clear history" 버튼 (우측 상단) → 전체 히스토리 삭제

## 예상 UX 흐름

```
1. 사용자가 "task:important" 검색
2. Task 클릭 후 페이지 이동
3. 다시 Cmd+K 누르고 포커스
4. 검색창이 비어있으면 "Recent" 섹션에 "task:important" 표시
5. 그 항목 클릭 → "task:important" 자동 입력 + 즉시 검색 실행
```

## 이점

| 항목 | 설명 |
|------|------|
| **작업 효율성** | 같은 검색을 반복할 때 타이핑 시간 단축 |
| **인지 부하** | 자주 찾는 항목이 한눈에 보임 |
| **발견성** | 사용자 검색 패턴의 가시화 |
| **접근성** | 키보드 네비게이션과 통합 (↑↓ 지원) |

## 기술 고려사항

### 보안
- `localStorage`에 민감 정보(비밀번호 등) 저장 안 함
- 검색어는 공개 정보 (Task ID, Doc 제목)

### 성능
- 히스토리 최대 5개 제한 → 메모리 오버헤드 최소
- localStorage 접근은 동기 → 마이크로태스크로 분리 가능 (선택사항)

### 호환성
- `localStorage` API는 모든 모던 브라우저 지원
- 비활성화 환경(private mode) → 조용히 실패 처리

## 데이터 유지 기간

- 수동 삭제까지 무기한 유지
- "Clear history" 버튼으로 사용자가 언제든 초기화 가능
