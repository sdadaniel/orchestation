# GlobalSearch 최근 검색어 히스토리 기능 제안

- **관련 태스크**: TASK-338
- **작성일**: 2026-04-03
- **상태**: 제안

## 배경

GlobalSearch 컴포넌트의 타입 안전성 수정 중 발견된 UX 개선 기회.
현재 검색창은 매번 새로 입력해야 하며, 반복적으로 동일 태스크/문서를 검색하는 패턴이 존재함.

## 제안 내용

### 최근 검색어 히스토리

| 항목 | 설명 |
|------|------|
| 저장소 | `localStorage` (`global-search-history` 키) |
| 최대 개수 | 5개 (FIFO) |
| 표시 조건 | 검색창 포커스 + 입력값 없음 |
| 동작 | 항목 클릭 시 해당 검색어로 즉시 검색 실행 |
| 삭제 | 각 항목에 X 버튼으로 개별 삭제 가능 |

### 구현 범위

- `src/frontend/src/components/GlobalSearch.tsx` — UI 및 localStorage 연동
- 별도 hook 추출 불필요 (단일 컴포넌트 내 로직)

### UI 설계

```
┌─────────────────────────────┐
│ 🔍 검색...                   │
├─────────────────────────────┤
│ 최근 검색                    │
│  TASK-298              ✕    │
│  cost parser           ✕    │
│  DAGCanvas             ✕    │
└─────────────────────────────┘
```

- 최근 검색어 영역은 `text-muted-foreground` 스타일
- 기존 검색 결과 목록과 동일한 키보드 네비게이션 지원

### 미구현 사항 (향후 고려)

- 검색 빈도 기반 정렬 (현재는 시간순)
- 서버 동기화 (현재는 로컬만)
